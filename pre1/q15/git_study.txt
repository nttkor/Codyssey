Git 명령어를 사용한 실습은 Git을 효과적으로 사용하기 위한 필수적인 과정입니다. 다음은 기본적인 Git 명령어들을 사용한 실습 예시입니다. 
1. 저장소 초기화:
git init:
현재 디렉토리를 Git 저장소로 초기화합니다. 이 명령어를 실행하면 .git이라는 숨김 폴더가 생성되어 Git 저장소 관련 파일들을 관리합니다.
git clone <원격 저장소 주소>:
원격 저장소에 있는 코드를 로컬로 복제합니다. 예를 들어, git clone https://github.com/사용자/저장소.git 와 같이 사용합니다. 
2. 파일 관리:
git status:
현재 저장소의 상태 (수정된 파일, 추가된 파일 등)를 확인합니다.
git add <파일 이름>:
변경된 파일을 스테이징 영역에 추가합니다. git add . 명령어를 사용하면 모든 변경된 파일을 한 번에 추가할 수 있습니다.
git commit -m "커밋 메시지":
스테이징 영역에 있는 변경 사항들을 커밋합니다. 커밋 메시지는 변경 내용을 간략하게 설명하는 것이 좋습니다. 
3. 원격 저장소와 연동:
git remote add origin <원격 저장소 주소>: 로컬 저장소를 원격 저장소에 연결합니다. 
git push origin <브랜치 이름>: 로컬 저장소의 내용을 원격 저장소로 업로드합니다. 일반적으로 git push origin main 또는 git push origin master 와 같이 사용합니다. 
git pull origin <브랜치 이름>: 원격 저장소의 변경 내용을 로컬 저장소로 가져옵니다. 
4. 브랜치 관리:
git branch: 로컬 브랜치 목록을 확인합니다.
git branch <새로운 브랜치 이름>: 새로운 브랜치를 생성합니다.
git checkout <브랜치 이름>: 지정된 브랜치로 전환합니다.
git merge <병합할 브랜치 이름>: 현재 브랜치에 지정된 브랜치를 병합합니다. 
실습 예시:
새로운 폴더를 생성하고 해당 폴더로 이동합니다.
git init 명령어를 사용하여 저장소를 초기화합니다.
텍스트 파일을 생성하고 git add 명령어를 사용하여 스테이징 영역에 추가합니다.
git commit 명령어를 사용하여 파일을 커밋합니다.
원격 저장소를 추가하고 git push 명령어를 사용하여 로컬 저장소의 내용을 원격 저장소로 업로드합니다.
다른 브랜치를 생성하고 전환하여 작업 후, 다시 main 또는 master 브랜치로 병합하는 과정을 연습합니다.
