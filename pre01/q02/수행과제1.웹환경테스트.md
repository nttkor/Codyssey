[ 1. 웹 환경 테스트 과제제출 - app.py](./app.py) Flask를 사용하여 간단한 메시지를 출력하는 웹 애플리케이션
```python
from flask import Flask # Flask 모듈을 가져옵니다.
app = Flask(__name__)   # Flask 애플리케이션 객체를 생성합니다.
@app.route('/') # 루트 URL에 대한 라우트를 정의합니다.
def hello_world(): # 이 함수는 루트 URL에 접근했을 때 호출됩니다.
    return "Hello, DevOps!" # "Hello, DevOps!" 메시지를 반환합니다.
# app.run(debug=False,host='0.0.0.0', port=80)  # This will run the Flask application not in debug mode 
app.run(debug=True,host='0.0.0.0',port=80)  # This will run the Flask application in debug mode on
```
- ##코드해석
```python
from flask import Flask
```
Flask 클래스(웹 애플리케이션 생성기)를 임포트합니다.
Flask는 Flask 프레임워크의 핵심 객체입니다.

```python
app = Flask(__name__)
```
Flask 애플리케이션 인스턴스를 생성합니다.
__name__은 현재 모듈의 이름으로, Flask가 애플리케이션의 위치를 파악하는 데 사용됩니다.
이 인스턴스 app이 웹 서버 역할을 합니다.

```python
@app.route('/')
```
Flask의 라우팅 데코레이터입니다.

'/'는 웹사이트의 루트 주소 (예: http://localhost:5000/)를 의미합니다.

사용자가 해당 주소로 요청을 보내면, 아래 함수가 실행됩니다.

```python
def hello_world():
    return "Hello, DevOps!"
```
루트 주소로 접속했을 때 브라우저에 "Hello, DevOps!"라는 문자열을 응답합니다.

실제 웹 페이지에 이 문자열이 표시됩니다.

전체 요약
이 코드는 Flask 웹 서버를 생성하고, 웹사이트의 루트 URL(/)에 접속했을 때 "Hello, DevOps!"라는 메시지를 보여주는 가장 기본적인 웹 애플리케이션입니다.

## Flask `debug=True`와 `debug=False`의 차이점

### ? `debug=True`일 때 (디버그 모드 ON)

- **코드 자동 리로드**
  - 코드가 변경되면 서버가 자동으로 재시작됨
- **디버깅 화면 제공**
  - 에러 발생 시 브라우저에서 자세한 에러 정보와 traceback 확인 가능
  - 웹에서 변수 값 확인 및 코드 실행 가능 (Werkzeug Debugger)
- **개발 편의 기능 제공**
  - 자세한 로그 출력 등
- ?? **보안 위험**
  - 운영 환경에서 사용 시 내부 정보 노출 및 악의적인 접근 가능

---

### ? `debug=False`일 때 (디버그 모드 OFF)

- **자동 리로드 없음**
  - 코드 변경 후 수동으로 서버 재시작 필요
- **간단한 에러 페이지**
  - 브라우저에는 에러 코드만 표시됨 (예: 500 Internal Server Error)
- **보안상 안전**
  - 운영 환경에 적합

---

### ? 요약 표

| 설정         | 설명                                       | 권장 환경       |
|--------------|--------------------------------------------|----------------|
| `debug=True` | 자동 리로드, 디버깅 도구, 에러 메시지 제공 | 개발 환경 (Dev) |
| `debug=False`| 안전한 실행, 디버깅 기능 없음              | 운영 환경 (Prod)|

---

> ? 운영용으로 배포할 경우 `debug=False`로 설정하고, `gunicorn` 또는 `uwsgi`와 같은 WSGI 서버를 사용하는 것을 권장합니다.


- 0.0.0.0의 뜻
  '0.0.0.0'의 뜻은 '지정된 주소가 없음'이란 의미며, 쉽게 '모든 주소'라고 해석해도 된다
  '0.0.0.0'은 라우터에서 '디폴트 라우트 정책'에 쓰인다
  '디폴트 라우트'란 정책에 해당되지 않는 패킷들이 들어왔을 때 기본적으로(default) 보낼 인터페이스(길, route)을 만들어두는 정책이다.
  다른 컴퓨터가 접속하는 '서버' 컴퓨터에서도 '0.0.0.0'은 '지정된 주소 없음, 모든 주소'란 의미지만, '로컬(컴퓨터와 연결된) 주소'에서의 '0.0.0.0'은 로컬 주소이기에 '내 컴퓨터랑 연결된 모든 주소'가 된다
  DHCP 서버에게 동적으로 IP주소를 할당받는 컴퓨터들은 IP주소를 할당받지 못했을 때 '0.0.0.0' IP주소를 갖는다, '윈도우즈 운영체제' 컴퓨터인 경우 '169.254.x.x/16' 주소를 갖는다.
- 127.0.0.1과의 차이
  사실 이미 위에서 0.0.0.0의 용도를 보면 알겠지만, 그냥 딱 짚어서 말하자면 '127.0.0.1'은 자기 자신을 가리키는 용도, 그 딱 한개뿐이다.
  하지만 '0.0.0.0'은 위에서 봤다시피  '특정한 주소를 지정하지 않음, 모든 주소'라는 의미로 다양하게 쓰인다.

- 포트 번호는 네트워크 통신에서 특정 애플리케이션이나 서비스를 식별하는 데 사용되는 고유한 번호입니다. 
  포트 번호 충돌은 여러 애플리케이션이 동일한 포트 번호를 사용하려고 할 때 발생하며, 이 경우 해당 애플리케이션은 정상적으로 동작하지 않을 수 있습니다. 충돌 해결 방법으로는 사용 중인 포트 번호 확인 후 사용하지 않는 번호로 변경하거나, 이미 사용 중인 프로세스를 종료하는 방법 등이 있습니다. 

- 포트 번호의 의미:
  네트워크 통신의 식별자:
  포트 번호는 IP 주소와 함께 사용되어 네트워크 상에서 특정 애플리케이션이나 서비스를 구분하는 역할을 합니다.
  애플리케이션 식별:
  동일한 IP 주소 내에서 여러 애플리케이션이 동시에 실행될 수 있는데, 이때 포트 번호를 통해 각 애플리케이션이 데이터를 주고받을 목적지를 식별합니다.
  잘 알려진 포트 번호:
  0 ~ 1023 번까지는 잘 알려진 서비스 (예: HTTP는 80번, HTTPS는 443번) 에 예약되어 있습니다.
  등록된 포트 번호:
  1024 ~ 49151 번은 등록된 포트 번호로, 특정 서비스에 할당될 수 있습니다.
  동적/사설 포트 번호:
  49152 ~ 65535 번은 동적 또는 사설 포트 번호로, 사용자가 임의로 사용할 수 있습니다. 
  포트 번호 충돌 시 해결 방안:
  1. 충돌 확인:
  netstat -a -o 명령어를 사용하여 현재 사용 중인 포트 번호와 해당 포트를 사용하는 프로세스의 PID (Process Identifier) 를 확인합니다. 
  lsof -i :포트번호 명령어를 사용하여 특정 포트를 사용 중인 프로세스를 확인할 수도 있습니다. 
  2. 충돌 해결:
  포트 번호 변경: 충돌하는 애플리케이션의 설정 파일에서 사용하지 않는 포트 번호로 변경합니다. 예를 들어, 톰캣의 기본 포트인 8080이 충돌한다면, 다른 포트 번호 (예: 8081) 로 변경합니다. 
  프로세스 종료: taskkill /f /pid PID 명령어를 사용하여 해당 PID를 가진 프로세스를 종료합니다. 단, 이 방법은 신중하게 사용해야 하며, 필요한 경우 서비스 재시작 등으로 인해 다른 문제가 발생할 수 있습니다. 
  방화벽 설정 변경: 필요에 따라 방화벽 설정을 변경하여 특정 포트 번호의 접근을 허용하거나 차단할 수 있습니다
